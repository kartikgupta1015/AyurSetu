# AyurSetu: The 100% Offline Vernacular Health Intelligence Platform

This is the official repository for the **Codecure @ SPIRIT 2026, IIT (BHU) Varanasi** hackathon submission by **Kartik Gupta (kartikgupta1015)**.

## 🏆 The Hackathon Challenge We Solved
India faces three massive, interconnected healthcare crises that standard telemetry apps completely fail to address:
1. **The Misinformation Epidemic:** Health rumors and fake treatments spread like wildfire on WhatsApp, primarily in vernacular regional languages that modern AI detectors struggle to read.
2. **Rural Connectivity:** The most vulnerable populations live in areas with 2G or zero internet access. Most health-tech "AI" requires heavy REST API calls to OpenAI or Claude, making them utterly useless in the field.
3. **The Disconnected Ecosystem:** Health apps are built for patients, leaving ASHA (Accredited Social Health Activist) workers completely blind to local disease clustering.

## 🚀 Introducing AyurSetu
We pivoted away from standard LLM cloud calls to build the ultimate **100% Offline Local Inference Engine**. AyurSetu gives rural citizens zero-latency clinical triage and myth-busting without spending a single byte on external API traffic.

### Core Features (The "God Mode" Ecosystem)
1. **Offline WhatsApp Myth Buster:** A blazing-fast `fuse.js` fuzzy-search engine securely embedded with 54+ highly verified ICMR/WHO health fact-check ontologies. Instantly slaps a "SAHI" (TRUE) or "GALAT" (FAKE) verdict on forwarded rumors without needing internet.
2. **Speak-to-Triage (Voice API):** Rural users who cannot type can click the floating microphone. By reverse-hooking Chrome's native `SpeechRecognition` API, they can verbally describe their symptoms in Hindi, Tamil, Telugu, Marathi, or English.
3. **Deterministic Clinical AI:** Replaced laggy LLM calls with a rigid, high-speed CSV-based multi-ontology ruleset that deterministically parses 41+ global medical symptom conditions to assign an exact Severity (Level 1-5).
4. **ABHA Identity Injection:** Direct mock-integration showcasing Ayushman Bharat Health Account (ABHA) connectivity. Linking your 14-digit ID adjusts the triage slip based on simulated offline government medical records (e.g. Diabetics get higher triage scores).
5. **ASHA Watchtower Command Center:** A dual-sided platform. Navigating to `/asha-dashboard` loads the ASHA worker's dark-mode surveillance radar. It logs every Level 5 "Red Alert" ping generated by villagers so workers can physically track dengue or rabies outbreaks.

## 💻 Tech Architecture

Our codebase is a highly optimized monorepo designed to compile instantly:
*   **Web Frontend (Dashboard UI):** Next.js 15 (App Router), TailwindCSS v4. Heavy use of custom keyframe animations and pure CSS glassmorphism for a "God Mode" aesthetic.
*   **Local Processing Backend:** Node.js 20 + Fastify running on port 3002.
*   **Search Engine:** Local `fuse.js` instance injected with precompiled `.csv` datasets instead of an external Vector DB. No Supabase. No PostgreSQL. Pure Edge speed.

## 🛠️ How to Boot the Project Locally

Because we engineered this to be hyper-lightweight for the rural edge, booting the ecosystem takes 5 seconds:

**1. Install Dependencies**
```bash
npm install
```

**2. Boot the Edge Medical Backend**
Starts the Fastify engine that handles the fuzzy inference logic.
```bash
npm run dev:backend 
```

**3. Boot the Next.js Patient Dashboard**
Open a second terminal window.
```bash
cd apps/web
npm run dev
```

**4. Demo the App**
Open your browser to `http://localhost:3000`. 
* Click the dropdown to switch to Hindi/Tamil/Telugu/Marathi language strings.
* Click the Microphone icon to test offline voice dictation!
* Input `91-1234-5678` into the ABHA Link module to test Ayushman Bharat integration.
* Use the floating bottom-right **ASHA Portal** button to warp into the surveillance Node.
